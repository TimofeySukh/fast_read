# Пользовательский поток

## Основной поток
1. Пользователь открывает сайт.
2. Видит `Upload PDF`, `Select Speed`, переключатель темы.
3. Загружает PDF.
4. Выбирает скорость в WPM или оставляет default `100`.
5. Нажимает запуск.
6. Видит экран обратного отсчета 3..1 (только цифры по центру).
7. Читает текст в режиме показа по одному слову (до пробела).
8. Каждую минуту чтение автоматически останавливается на 3 секунды.
9. После завершения текста открывается форма фидбека.
10. Заполняет nickname/email.
11. Указывает оценку скорости: too slow / normal / too fast / custom.
12. Добавляет свободный feedback.
13. Отправляет форму; данные сохраняются.
14. Видит экран «Спасибо за уделенное время» с таблицей итогов отправки.

## Альтернативные потоки
- Пользователь не меняет скорость: используется значение по умолчанию.
- Пользователь выбирает вариант `custom` и вводит свою оценку скорости вместо фиксированного варианта.

## Edge Cases
- PDF не удалось прочитать или в нем нет пригодного текста.
- Пользователь прерывает сессию до конца текста и не доходит до формы.
- Пользователь отправляет форму с неполными обязательными полями.
