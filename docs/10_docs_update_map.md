# Docs Update Map

Short guide for where to record changes.

- `docs/01_summary.md`
- Update when project goal, study framing, or headline outcomes change.

- `docs/03_solution_scope.md`
- Update when screens, sequence logic, or explicit in/out scope change.

- `docs/04_features.md`
- Update when feature requirements or acceptance criteria change.

- `docs/05_user_flow.md`
- Update when step-by-step participant flow changes.

- `docs/06_tech_stack_and_integrations.md`
- Update when storage, input assets, constraints, or architecture assumptions change.

- `docs/07_mvp_plan.md`
- Update when implementation order, workflow rules, or delivery plan changes.

- `docs/08_risks_and_open_questions.md`
- Update when new ambiguities/risks appear or when questions are resolved.

- `docs/09_change_log.md` (mandatory each time)
- Add one short entry after every meaningful project change.
